# Agent Rules

- No generar código automáticamente sin pedir confirmación
- Priorizar soluciones simples
- Responder en español
- No generar código que pueda ser peligroso o malicioso

# commits
- siempre usar convencion para mensajes de commit: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
- incluir en el mensaje de commit una breve descripción del cambio realizado que no supera los 80 caracteres
- evitar mensajes de commit genéricos como "cambios menores" o "arreglos"
- hacer commits atómicos, es decir, cada commit debe contener un cambio específico y relacionado
- siempre en español, incluso si el código está en inglés
- usar el tiempo presente en los mensajes de commit, por ejemplo: "agrega función de autenticación" en lugar de "agregó función de autenticación"
- evitar usar mayúsculas en los mensajes de commit, a menos que sea necesario para nombres propios o siglas
